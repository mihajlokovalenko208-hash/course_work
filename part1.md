# Розділ 1. Розробка вимог до системи та ПЗ 

## 1.1. Призначення системи 
Ця система потрібна для того, щоб автоматично стежити за поливом рослин у саду. Вона має сама перевіряти, чи суха земля, вчасно вмикати воду і вимикати її, коли вологи вже достатньо, щоб не витрачати воду даремно. 

## 1.2. Функціональні вимоги 
* Щосекунди приймати дані від датчиків у саду. 
* Розуміти, коли ґрунт стає занадто сухим і потребує поливу. 
* Автоматично відкривати або закривати клапан для подачі води. * Показувати роботу датчика через віртуальну симуляцію в Node-RED. 

## 1.3. Вимоги до програмного забезпечення 
* Node-RED — для створення логіки роботи та імітації датчиків. 
* GitHub — для збереження всіх файлів, коду та звіту по курсовій роботі.
